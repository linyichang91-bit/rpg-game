## Bug 修复: 创世界面不能滚动 + SSE 流双重 read 导致回合中断

### 创世界面滚动修复
- **问题**: `.app-shell` 固定 `height: 100dvh; overflow: hidden`，创世界面内容超出视口后无法滚动，底部按钮不可见
- **修复**: 新增 `.app-shell.is-genesis` 修饰类，创世模式下设为 `height: auto; min-height: 100dvh; overflow: visible`，允许页面撑高滚动
- `sandbox-root.tsx` 根据 `gameState` 有无动态添加 `.is-genesis` 类
- `html`/`body` 使用 `:has(.app-shell.is-genesis)` 条件切换 `overflow`，创世模式允许滚动，游戏模式保持 `overflow: hidden`
- 文件: `app/globals.css`, `components/sandbox-root.tsx`

### SSE 流双重 read bug（根因修复）
- **问题**: `consumeTurnStream` 的 while 循环中 `reader.read()` 被调用了两次——一次在 `Promise.race` 中（用于 abort 检测），一次在循环体中。第一次 read 的结果被丢弃，第二次 read 可能读到空 chunk 或 done=true，导致 `turn.completed` 事件被跳过，触发"流式连接在回合完成前中断了"错误
- **修复**: 删除 `Promise.race` + `abortWatcher` 模式，改为在每次 `reader.read()` 前直接检查 `options.signal?.aborted`，只做一次 read
- 文件: `lib/api.ts`

### app.py except 缩进修复
- **问题**: `event_stream()` 中 except 块缩进与 try 不对齐
- **修复**: 合并 `LLMGatewayError` 和 `asyncio.TimeoutError` 为单个 except 块，修正缩进
- 文件: `server/api/app.py`
